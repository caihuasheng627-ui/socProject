## 为什么会做这次修复

旧流程中主要有以下问题：

1. LSTM 实际看到的数据截止到决策日前一个观测，但交易使用决策日价格，预测跨度错了一步。
2. 旧模型比较曾用 `current_price` 计算回归误差，而模型预测的是未来价格。
3. Hybrid 的选择和最终评价都使用 val，没有用独立 test 做最后检验。
4. 回测曾根据最终收益反向压缩历史收益曲线，这种结果无法真实复现。
5. 深度模型和树模型使用的日期范围不同，却直接比较累计收益。
6. 旧树模型 CSV 只有 4 列，不能确认 split、目标日期和未来真实价格是否一致。

这些问题不代表原来的工作没有价值。模型、特征和主要代码都可以继续使用，主要需要统一“模型到底在预测什么”和“结果怎么比较”。

## 提议采用的统一预测规范

```text
决策观测：t
输入窗口：t-59 ... t，共 60 个观测，包含决策观测
预测目标：同一物品后续第 7 个有效日频观测
```

预测 CSV 统一为：

```text
split,date,target_date,market_hash_name,current_price,
actual_future_price,predicted_price,horizon_steps
```

其中：

- val 用于早停、模型选择和 Hybrid 路由选择。
- test 只用于最终评价，不参与选模型。
- 所有进入主表的模型必须使用完全相同的 test 样本。

## 我本地已经准备的改动

### 深度模型

- LSTM-C 窗口改为包含决策观测。
- LSTM-C 增加 `__UNK__`，用于训练集未见的新物品。
- LSTM-D 的价格分组边界只从 train 计算。
- LSTM-D 新物品按决策日当前价格分组，不读取整个 val/test 时段价格。
- LSTM-C、LSTM-D×3、GRU 已按新规范重训。

### 树模型

- RF、LightGBM、XGBoost 保留原来的默认参数，没有重新大范围搜参。
- val 预测只使用 train 拟合。
- test 预测使用 train+val 拟合。
- 新模型包保存模型、train-only 类别编码器、23 列特征顺序和预测信息。
- 已生成 RF、LightGBM、XGBoost 的 val/test 规范 CSV。

### 模型比较和回测

- 回归真值统一改为 `actual_future_price`。
- 六个主模型统一使用 35,229 条、154 件 test 样本。
- 删除回测中的事后收益 cap。
- 统一回测日期和物品范围。
- 增加零手续费和 2.5% 手续费两种结果。
- 当前自动测试为 `22 passed`。

## 新结果说明

完整 test 主表如下：

| 模型 | RMSE | MAE | MAPE | R² |
|---|---:|---:|---:|---:|
| **LSTM-C** | **44.3745** | **6.2668** | 10.95% | **0.9374** |
| LSTM-D | 52.0901 | 7.0234 | 7.72% | 0.9137 |
| Hybrid | 52.0900 | 7.0204 | 10.69% | 0.9137 |
| **RF** | 52.8212 | 7.2600 | **6.26%** | 0.9113 |
| LightGBM | 58.3566 | 8.4611 | 6.34% | 0.8917 |
| XGBoost | 61.7568 | 9.1675 | 7.93% | 0.8787 |

这组结果说明：

- LSTM-C 的 RMSE、MAE、R²最好。
- RF 的 MAPE 最低。
- Hybrid 在 val 上选出的最终路由是 `low→C、mid/high→D`。
- Hybrid 没有在 test 上全面超过 LSTM-C，所以不建议继续写“Hybrid 全指标严格最优”。

test 明显比 val 难，深度模型和树模型都出现指标下降。这更像是测试时段市场变化，而不是某一个模型单独失效。

GRU 只覆盖 top10，ARIMA 只覆盖少数代表物品，建议继续放附表，不与覆盖 154 件的六个模型直接排名。

## 是否有必要采用

可以考虑三个方案：

### 方案 A：不采用

继续使用旧模型、旧预测 CSV 和旧指标。

- 优点：不需要改后端。
- 问题：预测窗口、未来真值和回测公平性问题仍然存在，答辩时难以解释。

### 方案 B：只更新报告数字

保留旧部署，只把部分指标换成新结果。

- 优点：改动较少。
- 问题：报告和系统实际使用的模型口径不同，不建议采用。

### 方案 C：完整采用（推荐）

同时合入预测规范、模型、预测文件、比较/回测和后端兼容。

- 优点：训练、评价和在线推理口径一致，结果可以解释和复现。
- 成本：组员 3 需要做一轮后端适配，组员 2 需要确认树模型参数和脚本合并方式。
