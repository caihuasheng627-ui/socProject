# 文献精读 #2 — Liang et al. 2024:多 Agent 辩论中的"分歧"导向

> 组员 3 精读稿 · 按 5 段模板(方法 / 优势 / 局限 / 与本项目关系 / 可迁移点)
> 论文主题:在多智能体辩论中鼓励发散性思维(Divergent Thinking in Multi-Agent Debate)
> 注:引用细节以组员 1 汇总稿为准,本文聚焦可迁移结论。

## 1. 方法

该工作研究多 Agent 辩论框架中"收敛过快"的问题:经典辩论设定(如 Du et al. 2023)让多个 LLM Agent 逐轮互相质疑、最终趋向共识,但实践中 Agent 往往在前 1-2 轮就因"从众"而快速收敛,丢失了多元视角。论文提出在辩论流程中显式注入"分歧"机制——通过角色异构化(不同 Agent 持不同先验/不同评估维度)、轮间立场扰动、或对"少数派观点"的奖励,迫使 Agent 在达成共识前保留并表达差异化的推理路径,再在最后阶段做有据可依的收敛。实验在推理与决策任务上对比了"纯收敛辩论"与"先发散后收敛"两种流程。

## 2. 优势

- **避免 premature convergence**:多 Agent 辩论最大的价值是视角多样性;过早共识等于退化为单模型,该工作把"保持分歧"作为一等目标,保留了辩论相对单模型的核心增益。
- **角色异构化带来可控的偏置**:让 Bull/Bear、技术派/基本面派等角色携带不同先验,比"同质 Agent 多次采样"更能覆盖相反假设。
- **可解释性**:分歧-收敛的过程本身就是一段可读的推理轨迹,便于人工审视模型在哪一轮、因何种证据改变立场。

## 3. 局限(不能直接照搬到饰品市场)

- **任务领域差异**:原论文在通用推理/问答任务上验证,而饰品价格是噪声极大的弱信号时序,Agent 容易把"发散"演变成"各说各话",在没有扎实量化锚点时难以收敛到可操作结论。
- **成本与延迟**:多轮 × 多 Agent 的 LLM 调用在实时接口下延迟与费用可观,课程演示场景难以每请求都现场重跑。
- **缺乏量化收敛判据**:论文的收敛多为"轮次到即止",没有给出"何时分歧已足够、可以收敛"的定量标准,直接搬到交易决策会显得主观。

## 4. 与本项目关系

SkinVision AI 的 `/api/debate/{skinId}` 正是双 Agent(🐂 Bull vs 🐻 Bear)3 轮辩论结构,且组员 3 在方案 B 中明确设计了**预录回放 / 现场重跑双模式**:默认走预录 JSON(规避成本与延迟,Expo 1 秒展示),评委要求时切 `?live=1` 现场重跑。这一设计直接回应了本文指出的"成本/延迟"局限。同时本项目的辩论每轮都**以 Hybrid 模型的量化预测(current price、7 天预测价、置信度)为锚点**,而非纯文本推理,缓解了"各说各话不收敛"的问题——Bull/Bear 的立场必须落到具体的目标价/止损位上。

## 5. 可迁移点(Opportunities for innovation)

- **分歧度量化**:借鉴本文,在辩论输出里加一个"分歧度"指标(Round 1→3 立场距离的衰减曲线),作为 `consensusScore` 之外的补充信号——分歧衰减过快可能提示 Agent 从众,衰减过慢可能提示信号不足。落到 **Agent 辩论** 模块。
- **角色异构化扩展**:当前只有 Bull/Bear 两角色;可按本文思路扩展为"技术派(RSI/MACD)/ 基本面派(Major 赛程/供给)/ 流动性派(成交量/买卖盘)"三方,先发散再两两收敛,提升覆盖。落到 **Agent 辩论** 模块。
- **先发散后收敛的两阶段 prompt**:Round 1-2 鼓励分歧(显式要求"提出对方未考虑的风险"),Round 3 才要求共识,与本项目现有 3 轮结构天然对齐。落到 **Agent 辩论** 的 prompt 工程。
- **与组合诊断联动**:把单件辩论的分歧度作为该物品的"观点分歧风险"因子,纳入 **组合诊断** 的风险贡献 Top N,让"市场分歧大"的持仓被优先提示。
